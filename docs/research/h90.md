# Eventide H90 ‚Äî MIDI control research

Research backing `internal/device/definitions/h90.yaml`. Firmware reference:
**1.7.1** (manual at `cdn.eventideaudio.com/manuals/h90/1.7.1`).

## TL;DR

- **Programs** are recalled with **standard MIDI Program Change**. Default PC
  range is **1‚Äì100**; enable **PC Offset** in the System Menu to use **0‚Äì99**.
- There is **no MIDI Bank Select (CC0/CC32)**. "Bank Up/Bank Down" are
  CC-mappable *navigation* functions (they move ¬±3 Programs), not Bank Select
  messages. You also **cannot change the Playlist over MIDI**.
- The H90 has **zero default MIDI CC assignments**. Every CC function must be
  user-assigned on the device (**System Menu ‚Üí MIDI ‚Üí Global Control**) or in
  H90 Control. The CC numbers in the YAML are therefore a **server-side
  convention**, not Eventide factory values ‚Äî the H90 must be configured to
  match them.
- **Switch semantics:** Active/Bypass + trigger functions act on the CC value:
  **`>= 64` = active/engage**, **`< 64` = bypass/off**.
- **No documented public SysEx parameter API.** Deep per-algorithm parameter
  control is done via the Quick Knob / HotKnob / Performance global mappings
  (which are CC) or the H90 Control app. No SysEx controls were invented.

## MIDI channel behaviour (S1)

- **Channel:** user-selectable 1‚Äì16; the H90 receives *and* transmits on this
  channel.
- **Receive Omni:** when on, the H90 receives on all channels simultaneously
  (overrides the receive channel); it still transmits on the configured channel.
- **Output Mode:** `Off` / `Transmit` (sends PC, CC, MIDI Clock out the DIN
  out; no thru) / `Thru` (passes DIN-in to DIN-out; H90 sends nothing).
- The channel is **not** stored in the device definition ‚Äî per the project
  design it is supplied by the *binding* `(endpoint, channel, definition)`.

## Program recall (S1, S3, S4)

- Send **Program Change** to load a Program from the **currently selected
  Playlist**. PC 1 ‚Üí Program 1, PC 3 ‚Üí Program 3, etc.
- A Program is a pair of algorithm **presets**; you **cannot** select an
  individual preset over MIDI ‚Äî build Programs ahead of time in the order you
  need (S4).
- **PC Offset** (System Menu): default off ‚Üí PC **1‚Äì100**; on ‚Üí PC **0‚Äì99**
  (use when other gear counts programs from 0) (S1, S4).
- **Avoid sending PC and CC simultaneously** ‚Äî one may be ignored; hence the
  `settle_ms` delay between program change and CC overrides on recall (S4).

## No factory CC map ‚Äî Global Control (S1, S2)

> *"The H90 has no default MIDI CC or external controller mappings, all mappings
> are customizable by the user."* (S2)

Configure on the device: **System Menu ‚Üí MIDI ‚Üí Global Control** ‚Äî pick a
*Parameter* (Quick Knob 1) and a *CC #* (Quick Knob 2) (S1). The full list of
mappable parameters is the **Global Mapping Chart, Appendix C** (S2).

### Active/Bypass + trigger value rule (S2)

- `P Act/Byp` (toggle): a CC value **‚â• 64** toggles active/bypass state.
- `P Act/Byp (M)` (momentary): CC **< 64** ‚Üí bypass, CC **‚â• 64** ‚Üí active.
  The YAML uses the **(M)** semantics for deterministic state management
  (`bypass: 0`, `active: 127`).

## Confidence summary

- **High confidence (vendor-sourced):** `program` via Program Change, the
  1‚Äì100 / 0‚Äì99 PC Offset behaviour, the no-Bank-Select fact, the list of
  mappable functions, the `‚â•64`/`<64` switch rule, MIDI channel/Omni/Output Mode
  behaviour. All from S1/S2 (+S3/S4 forum confirmation).
- **Convention (NOT vendor defaults), must be configured on the H90:** every CC
  number. Function semantics are sourced; the *numbers* are a convention (C1 for
  the navigation/bypass/system block; invented-contiguous for the continuous
  params). Marked `# convention (no factory default)` in the YAML.

## Caveats

- All CC numbers require manual setup on the H90 (Global Control) or H90 Control;
  out of the box none of the CC controls do anything.
- `tuner` is toggle/display-side only ‚Äî engaging it shows the tuner on the H90's
  screen and does not transmit tuning data back over MIDI (S3).
- The forum threads (S3/S4) reference nearby firmware revisions (1.3.0, 1.7.5,
  1.9.x) for the same Global Mapping Chart; the chart is stable across these.
  Values were taken against **1.7.1** docs to match the project's pinned URL.
- No documented public SysEx for parameter editing was found; if a future need
  arises, capture it via MIDI-learn rather than guessing a SysEx frame.
